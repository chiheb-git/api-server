PAGE 3 — Add Phone (Core Innovation):
Input fields:

IMEI code (15 digits, no Luhn validation)
Photo of physical IMEI sticker
Brand (optional)
Model (optional)

When user clicks "Confirm & Analyze":
AI LAYER 1 — IMEI/Photo Correlation (patentable):
Extract the IMEI number from the sticker photo using visual pattern recognition. Compare extracted IMEI with manually typed IMEI. Calculate match percentage 0-100%.

Match 90-100% → IMEI verified ✅
Match below 90% → IMEI rejected ❌ → show "Photo and IMEI code do not match. Please try again."
Only if verified → proceed to Layer 2

AI LAYER 2 — Trust Score Calculation (patentable):
Calculate a dynamic trust score 0-100 based on:

IMEI/Photo match quality × 35 points
TAC code brand/model coherence × 25 points
Image quality and authenticity × 20 points
Registration metadata × 10 points
Geographic baseline × 10 points

Display animated circular gauge with:

🔴 0-40 = High Risk
🟡 41-70 = Medium Risk
🟢 71-100 = Verified Secure

AI LAYER 3 — Physical Forgery Detection (patentable):
Analyze the sticker photo pixels for:

Font inconsistency detection
Color anomaly detection
Edge variance analysis
Barcode/IMEI alignment check
Result: "No forgery detected ✅" or "Possible physical forgery ⚠️"

FINAL DECISION ENGINE:
Combine all 3 layers:

Layer 1 match + Layer 2 score above 70 + Layer 3 no forgery → SAVE to database → show "✅ Phone successfully added and verified"
Any layer fails → DO NOT SAVE → show specific error explaining which layer failed and why


PAGE 4 — Search Phone:
Input: IMEI code (15 digits)
When user clicks "Verify":
Search the database buffer for this IMEI.
IF IMEI EXISTS in database:

Show "⚠️ PHONE IS NOT SECURE 🔴"
Show "This device has been reported"

Show registration date and verification count

IF IMEI DOES NOT EXIST:

Show "✅ PHONE IS SECURE 🟢"
Show "This device has not been reported"
Show how many times this IMEI was searched
